## Bekannte Schwächen (bewusst stehen gelassen)

- **`is_safe_select` ist Heuristik, kein echter SQL-Parser**:
  Strings mit Escape-Quotes (`'don''t'`) werden falsch zerlegt,
  konservative Ablehnung als Konsequenz. Für ein Lernprojekt
  akzeptabel, ein Mini-Lexer wäre overkill.
- **`MAX_ZEILEN_FUER_CLAUDE = 50`** ist eine Bauchzahl, kein
  Token-Counting.
- **Connection und Cursor zwischen Threads geteilt**: nur
  unkritisch, weil Single-User-Demo. In einem Multi-User-Setup
  bräuchte es Connection-Pool oder Connection-pro-Thread.
- **Geld-Formatierung an Claude delegiert** statt
  deterministisch im Code – nicht reproduzierbar, kostet
  Tokens.
- **Inline-Orchestrierung in `app.py`** verhindert Headless-
  Integrationstests der zwei-Aufruf-Logik. Wird in Schritt 4
  durch Tool-Use natürlich obsolet.
- **Streamlit-File-Watcher abgeschaltet**: kein Auto-Reload
  bei Code-Änderungen mehr, Browser-Tab manuell neuladen.
- **pyarrow-Workaround rein dokumentarisch**: kein Test
  erzwingt das Fehlen von pyarrow.

Vollständige Liste mit Begründungen siehe Code-Review.
